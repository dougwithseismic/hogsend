# Hogsend marketing video workspace — conventions

Remotion v4 workspace for the Hogsend launch videos. Six videos are built
in parallel by six agents on top of this foundation. Read this whole file
before touching anything.

## The one hard rule

**Only ever edit your own `src/videos/<id>/` folder.** Everything else —
`src/lib/`, `src/components/`, `src/entries/`, `src/videos/index.ts`,
`src/Root.tsx`, `src/index.ts`, `src/fonts.ts` — is shared foundation and
owned by the foundation agent. If a render fails with an error pointing
into someone else's folder, **wait 60 seconds and retry — never fix it
yourself** (the other agent is mid-edit).

## Setup

```bash
# from marketing/video
pnpm assets        # copies screenshots/logos/fonts into public/ (gitignored)
pnpm studio        # Remotion Studio with ALL compositions
pnpm check-types   # tsc --noEmit — must stay green
```

Run `pnpm assets` once before rendering; `public/screenshots/` and
`public/logos/` are gitignored and regenerated by this script.

## Video registration contract

Each video lives at `src/videos/<id>/index.tsx` and exports exactly:

```tsx
import { defineVideo, type VideoProps } from "../../lib/define-video";

const MyVideo: React.FC<VideoProps> = ({ persona }) => { /* ... */ };

export const video = defineVideo({
  id: "my-video",          // matches the folder name
  durationInFrames: 360,   // fixed — do not change your budget
  fps: 30,                 // always 30
  personas: ["engineer"],  // only if briefed; defaultProps.persona = "engineer"
  component: MyVideo,
});
```

The registry (`src/videos/index.ts`) and `src/Root.tsx` already register
every video as three compositions: `<id>-169` (1920×1080), `<id>-916`
(1080×1920), `<id>-11` (1080×1080). Components receive `{ persona? }`
and derive EVERYTHING else from `useFormat()` — never hardcode pixel
positions that only work in one ratio.

Frame budgets (fixed): how-it-works **1200**, semantic-links **360**,
first-party-tracking **360**, byo-provider **360**, wait-for-event
**360**, scaffold-demo **450**.

### Isolated rendering

Each video has its own entry at `src/entries/<id>.ts` bundling ONLY that
video, so six agents can render in parallel without each other's code:

```bash
# still (fast visual check — do this constantly)
pnpm exec remotion still src/entries/<id>.ts <id>-169 out/check.png --frame=120

# draft video (fast, half scale)
pnpm exec remotion render src/entries/<id>.ts <id>-169 out/<id>-draft.mp4 --scale=0.5

# final (all three formats)
pnpm exec remotion render src/entries/<id>.ts <id>-169 ../renders/<id>-169.mp4
pnpm exec remotion render src/entries/<id>.ts <id>-916 ../renders/<id>-916.mp4
pnpm exec remotion render src/entries/<id>.ts <id>-11  ../renders/<id>-11.mp4

# persona variant
pnpm exec remotion render src/entries/how-it-works.ts how-it-works-169 \
  ../renders/how-it-works-founder-169.mp4 --props='{"persona":"founder"}'
```

Finals go to `marketing/renders/` (gitignored), drafts to `out/`.

## Brand

Tokens live in `src/lib/theme.ts` (`theme`, `typo`, `radius`, `syntax`)
— import them, never re-type hex values. They are EXACT ports of the
apps/docs design system (`app/global.css` + `components/ds`).

| token | value | use |
| --- | --- | --- |
| `theme.ink` | `#050101` | page background |
| `theme.paperPure` | `#0a0606` | cards, code mocks |
| `theme.accent` | `#f64838` | THE accent — ~one element per beat |
| `theme.accentDeep` | `#b8281c` | pressed/deep accent |
| `theme.accentTint` | `rgba(246,72,56,0.15)` | bands, pill fills |
| `theme.frameLine` | `white/0.07` | the two vertical gutter hairlines |
| `theme.hairlineFaint` | `white/0.08` | dividers, chip borders |
| `theme.cardBorder` | `white/0.1` | card/code-mock borders |
| `theme.hairline` | `white/0.15` | traffic lights, hover borders |
| `theme.text` … `theme.textGhost` | `#fff` → `white/0.2` | the docs text ladder (body 0.8, muted 0.6, faint 0.5, hint 0.4, ghost 0.2) |

Type metrics (`typo`): hero tracking `-0.055em` (Inter Display 500),
everything else `-0.02em`, eyebrows `0.08em` uppercase. Radii
(`radius`): card 6 / mock 10 / panel 12 (scaled in components).

Code uses the `syntax` palette — the EXACT github-dark Shiki colours the
site's code blocks render with (keyword `#f97583`, string `#9ecbff`,
func `#b392f0`, number `#79b8ff`, comment `#6a737d`, base `#e1e4e8`).
Never invent code colours; sample the site if a kind is missing.

Fonts (loaded by `src/fonts.ts`, blocks render until ready — never
hand-write font-family strings):

- `FONT_DISPLAY` — **Inter Display** (headlines; weights 400/500 only)
- `FONT_BODY` — **Inter** (body copy, eyebrows)
- `FONT_MONO` — **Geist Mono** (code, terminals, timestamps, status rows)

The look: near-black page with two faint vertical gutter hairlines
(never a closed box), film-grain noise, ONE red accent per beat,
generous emptiness. If a frame has two accent-coloured ideas, cut one.

**No title cards.** Open on the artifact — terminal, code, diagram —
with an `Eyebrow` above it the way the docs site opens a section. Let
typing/motion carry the first beat; save display type for one short
payoff line mid-video. Copy stays understated and factual (status rows
beat slogans). `EndCard` (on `glow glowPosition="bottom" dots`) closes
every video.

## Motion rules

- **Hard cuts and spring pops. Never crossfades, never eases-in-out
  between scenes.** Beat changes are cuts (optionally with `punchIn`).
- Springs: high damping (~200) via the presets in `lib/anim` —
  `pop`, `punchIn` (1.04→1.00 over the first frames of a beat),
  `slideUp`, `staggerWords` (3–4 frame word stagger).
- **Beats are 45–75 frames.** End cards hold ~60 frames.
- Nothing sits fully static longer than ~75 frames — `SceneShell`'s
  `drift` prop (or `microDrift`) adds the 1.00→1.02 hold drift.
- 30fps everywhere. Time everything in frames, not seconds.

### The beat system

Author a video as a list of timed beats; each beat is a hard-cut
`<Sequence>`:

```tsx
import { Beats, beat, totalFrames } from "../../lib/anim";

<Beats
  beats={[
    beat("hook", 60, () => (
      <SceneShell glow drift>
        <KineticText text="Journeys are *code*" size="xl" />
      </SceneShell>
    )),
    beat("proof", 75, () => (
      <SceneShell>
        <CodeScene filename="src/journeys/welcome.ts" code={SNIPPET} />
      </SceneShell>
    )),
    beat("end", 60, () => (
      <SceneShell glow>
        <EndCard />
      </SceneShell>
    )),
  ]}
/>
```

`totalFrames(beats)` must equal your `durationInFrames` — assert it
mentally before rendering.

## Formats and safe areas

`useFormat()` (from `lib/format`) returns
`{ ratio, width, height, isPortrait, pad, fontScale, safeTop, safeBottom }`:

- `ratio`: `"169" | "916" | "11"` — derived from `useVideoConfig()`.
- `pad`: base content padding inside the hairline frame.
- `fontScale`: multiply ALL font sizes by this (1 on 16:9).
- 9:16 keeps content in the centre-safe vertical band: `safeTop` /
  `safeBottom` reserve ~12% top/bottom for platform UI. `SceneShell`
  applies this automatically — extra absolute-positioned content must
  respect it too.
- Stack horizontally on 16:9, vertically on 9:16 (`FlowDiagram` does
  this for you; do the same in custom layouts via `isPortrait`).

Check every beat in all three ratios before calling it done — render a
still per ratio at each beat's midpoint.

## Component catalogue

All in `src/components/`. Everything is already responsive via
`useFormat()`; pass `width`/sizes only to override.

### SceneShell

Ink background + the docs PageFrame (two vertical gutter hairlines +
noise grain) + optional red aurora bloom + optional dot-grid backdrop +
optional hold drift. Wrap EVERY beat in one.

```tsx
<SceneShell glow glowPosition="bottom" dots drift>
  {children}
</SceneShell>
```

Props: `glow?` (aurora), `glowPosition?: "top"|"center"|"bottom"`,
`glowIntensity?` (0–1, animatable from the beat), `horizon?` (the hero's
red-rimmed planet — use on end cards), `dots?` (docs CTA dot grid),
`drift?`, `driftFrames?` (default 75), `justify?`/`align?`.

### Fx — Noise / Aurora / Horizon / DotGrid

The atmosphere layers SceneShell composes (use directly only for custom
stages): `<Noise opacity?>` film grain, `<Aurora position? intensity?
breathe?>` breathing red bloom, `<Horizon intensity?>` the hero planet,
`<DotGrid opacity?>` masked accent dots.

### Labels — Eyebrow / Kicker / GhostNumber / TagPill

The docs label language:

```tsx
<Eyebrow text="Durable waits" dot delay={2} />   // uppercase white/50 micro label
<Kicker text="Wait for the event." />            // accent-red intro line
<GhostNumber n="01" />                           // huge white/20 numeral
<TagPill text="deploy a3f9c12" mono />           // chip; `accent` for the red one
```

Open beats with an Eyebrow (negative `delay` renders it settled, for
continuity across hard cuts). One TagPill variant per beat.

### KineticText

Word-by-word spring headline in Inter Display. `*word*` renders accent.

```tsx
<KineticText text="Links that *answer* back" size="xl" delay={6} />
```

Props: `text`, `size?: "xl"|"lg"|"md"` (124/88/56 × fontScale),
`align?: "left"|"center"`, `delay?` (frames), `stagger?` (default 3),
`muted?`, `maxWidth?`.

### CodeScene

Editor card (paper-pure, hairline-faint border, mono traffic lights,
filename tab) that TYPES tokenized TypeScript with a block cursor and
line-end micro-pauses. Wrap ONE range in `⟦…⟧` for accent emphasis.

```tsx
<CodeScene
  filename="src/journeys/welcome.ts"
  code={`await ctx.sleep({ duration: ⟦days(2)⟧ });`}
  typeSpeed={2.4}
/>
```

Props: `code`, `filename?`, `typeSpeed?` (≈chars/frame, default 2.4),
`startDelay?`, `instant?` (skip typewriter), `footer?` (status row
below a hairline divider — reserve its height from frame 0), `width?`,
`fontSize?`. Use `typingDuration(code, typeSpeed)` from `lib/typewriter`
to size the beat so typing finishes with ~20 frames of hold.

### Terminal

Terminal card matching CodeScene chrome. Types the command after an
accent `❯` prompt, then reveals output lines (spinner→accent-tick tasks,
indented file trees).

```tsx
<Terminal
  command="pnpm dlx create-hogsend@latest"
  output={[
    { text: "Scaffolding your app", kind: "task", spin: 16 },
    { text: "src/journeys/welcome.ts", kind: "file", indent: 1 },
    { text: "Ready.", kind: "muted" },
  ]}
/>
```

Props: `command`, `output?: TerminalLine[]` (`kind: "line"|"task"|"file"|
"muted"`, `indent?`, `spin?`), `typeSpeed?` (default 1.6), `startDelay?`,
`lineStagger?` (default 7), `width?`, `fontSize?`, `title?`.

### BrowserChrome

Screenshot in minimal browser chrome (dots, url pill, hairline border)
with subtle Ken-Burns drift. `src` is relative to `public/`.

```tsx
<BrowserChrome src="screenshots/01-overview.png" url="hogsend.com/studio" />
```

Props: `src`, `url?`, `width?`, `still?`, `driftFrames?` (default 150).
Available screenshots: `01-overview`, `02-sends`, `04-journeys`,
`11-send-drawer`, `10-debug` (`.png`, after `pnpm assets`).

### EmailCard

Stylised email — real subject in display font, from-line, grey skeleton
body bars — sliding in, with a status row ticking
delivered → opened → clicked (accent ticks, mono timestamps).

```tsx
<EmailCard
  subject="Welcome to Hogsend"
  statusAt={{ delivered: 12, opened: 26, clicked: 40 }}  // frames
/>
```

Props: `subject`, `from?`, `statusAt?`, `times?` (timestamp strings),
`width?`, `bodyBars?: number[]` (bar widths 0–1).

### FlowDiagram

Chain of node chips (label and/or white-tinted logo) joined by
hairlines; an accent pulse dot travels the path and nodes pop as it
arrives. Horizontal on 16:9/1:1, vertical on 9:16.

```tsx
<FlowDiagram
  nodes={[
    { logo: "posthog.svg", label: "event" },
    { label: "journey" },
    { logo: "resend.svg", label: "send" },
  ]}
  pulseStart={8}
  hopFrames={18}
/>
```

Props: `nodes: { label?, logo? }[]`, `pulseStart?`, `hopFrames?`,
`scale?`. Logos in `public/logos/` (after `pnpm assets`): `hatchet`,
`posthog`, `railway`, `resend`, `segment`, `slack`, `stripe`,
`typescript` (`.svg`) — single-colour masks, rendered white via CSS mask.

### EndCard

Standard closer: Wordmark, one short line, the scaffold command in a
mono pill, hogsend.com. Give it a ~60-frame beat.

```tsx
<EndCard line="Lifecycle email, in your repo." />
```

Props: `line?`, `command?` (default `pnpm dlx create-hogsend@latest`),
`domain?` (default `hogsend.com`).

### Wordmark

The typeset brand lockup (accent tile + send glyph + "Hogsend" in Inter
Display medium) — replicates the docs-site nav. Not an image.

```tsx
<Wordmark size={72} />          // tile size in px; text scales with it
<Wordmark size={56} glyphOnly />
```

## Quality bar

Vercel/Linear launch-video tier. Before declaring a video done:

1. `pnpm check-types` green; `pnpm exec biome check marketing/video`
   green (from the repo root).
2. Stills checked at every beat boundary in all three ratios.
3. One accent element per beat, no crossfades, no static holds > 75
   frames, 9:16 content inside the safe band.
4. Draft mp4 watched end-to-end at least once.
