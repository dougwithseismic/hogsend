**Words you'll meet in this chapter**

- **hypothesis** — a change written so reality can prove it wrong: a change, an audience, an expected outcome, a metric, and the reasoning behind it.
- **primary metric (OEC)** — the single number, fixed before the test, that decides whether it won; Kohavi's Overall Evaluation Criterion.
- **guardrail metric** — a number the test isn't allowed to break, whatever it does to the primary — revenue, unsubscribe rate, latency.
- **statistical power** — the chance a test detects a real effect when one exists; low power both misses wins and manufactures false ones.
- **minimum detectable effect (MDE)** — the smallest change you care to catch; halve it and you need roughly four times the traffic.
- **p-value / statistical significance** — the odds a result this large is pure chance; "significant" means those odds sit below a line you set first (usually 5%).
- **holdout** — a slice of users kept out of the whole program, so the gap to everyone else reveals what the program actually added.
- **incrementality** — the outcome that happened because of the program and wouldn't have happened anyway.
- **ICE** — a backlog score — impact, confidence, ease, each 1–10 — used to order which experiment to run next.
