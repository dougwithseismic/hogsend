---
title: AI churn-save with Eve
description: A churn-risk journey starts an Eve session with fetch, parks on ctx.waitForEvent, and resumes when Eve POSTs its terminal result back via a defineWebhookSource receiver. No Eve dependency in the Hogsend worker — just fetch out and a signed webhook in.
---

Some retention decisions need a human. A customer who flags serious product pain doesn't want a template — they want someone to read their situation and decide. This recipe routes high-risk churners to an Eve agent session: Eve handles the HITL deliberation on its own infrastructure; the journey parks on `ctx.waitForEvent` and resumes when Eve signals completion.

The Hogsend worker has no Eve dependency. The integration is `fetch` out + `defineWebhookSource` in — two plain primitives over the existing webhook and event spines.

| Step | What does it |
|---|---|
| Churn risk detected | `churn.risk_flagged` event (emitted by your product or a scoring job) |
| Start Eve session | `fetch POST /eve/v1/session` with `metadata.userId` + `callbackEvent` |
| Journey parks | `ctx.waitForEvent({ event: AGENT_COMPLETED, timeout: days(7) })` |
| Eve POSTs back | `POST /v1/webhooks/eve` → `eveSource.transform()` → `ingestEvent()` |
| Journey resumes | parses the play, sends the save email if still subscribed |
| User reactivates mid-wait | `exitOn: [REACTIVATED]` → journey exits, no send |

## The Eve session helper

```ts
// src/lib/eve.ts
export interface EveSessionResult {
  sessionId: string;
}

export async function startEveSession(opts: {
  agent: string;
  userId: string;
  callbackEvent: string;
  input: Record<string, unknown>;
}): Promise<EveSessionResult> {
  const res = await fetch(`${process.env.EVE_BASE_URL}/eve/v1/session`, {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      Authorization: `Bearer ${process.env.EVE_TOKEN}`,
    },
    body: JSON.stringify({
      agent: opts.agent,
      input: opts.input,
      metadata: {
        userId: opts.userId,
        callbackEvent: opts.callbackEvent,
      },
    }),
  });

  if (!res.ok) {
    throw new Error(`Eve session start failed: ${res.status} ${res.statusText}`);
  }

  return res.json() as Promise<EveSessionResult>;
}
```

Eve's response carries a `sessionId` you can log for observability. The journey doesn't need it — it waits for the event, not the session.

## The webhook source

Eve POSTs its terminal result to `POST /v1/webhooks/eve`. The receiver verifies the HMAC signature and ingests the callback as a Hogsend event:

```ts
// src/webhook-sources/eve.ts
import { defineWebhookSource } from "@hogsend/engine";

export const eveSource = defineWebhookSource({
  meta: { id: "eve", name: "Eve Agent Callback" },
  auth: {
    type: "signature",
    scheme: "hmac-hex",
    header: "x-eve-signature",
    envKey: "EVE_WEBHOOK_SECRET",
  },
  transform(payload) {
    const userId        = payload?.metadata?.userId;
    const callbackEvent = payload?.metadata?.callbackEvent;
    if (typeof userId !== "string" || typeof callbackEvent !== "string") {
      return null; // malformed — drop silently
    }
    return {
      name: callbackEvent,           // e.g. "agent.completed"
      userId,
      eventProperties: {
        // Serialise the play as a string — event properties are scalars.
        play: JSON.stringify(payload.result ?? {}),
      },
    };
  },
});
```

Register it in `src/webhook-sources/index.ts`:

```ts
// src/webhook-sources/index.ts
import { eveSource } from "./eve.js";
export const webhookSources = [posthogSource, eveSource];
```

And pass `webhookSources` to `createApp`:

```ts
// src/index.ts (excerpt)
createApp(client, { webhookSources })
```

## The play schema

The journey parses Eve's typed result from the `play` event property:

```ts
// src/journeys/ai-churn-save.ts
import { z } from "zod";

const SavePlay = z.object({
  action:       z.enum(["send_offer", "escalate_to_human", "no_action"]),
  offerHeadline: z.string().optional(),
  offerDetail:   z.string().optional(),
});
```

## The journey

```ts
// src/journeys/ai-churn-save.ts
import { days, hours } from "@hogsend/core";
import { defineJourney, sendEmail } from "@hogsend/engine";
import { startEveSession } from "../lib/eve.js";
import { getUserContext } from "../lib/user-context.js";
import { Events, Templates } from "./constants/index.js";

export const aiChurnSave = defineJourney({
  meta: {
    id: "ai-churn-save",
    name: "AI Churn Save (Eve HITL)",
    enabled: true,
    trigger: {
      event: Events.CHURN_RISK_FLAGGED,
      // Only high-confidence churn signals.
      where: (b) => b.prop("score").gte(70),
    },
    entryLimit: "once",
    suppress: hours(12),
    // User reactivates mid-wait — no need to send the save email.
    exitOn: [{ event: Events.REACTIVATED }],
  },

  run: async (user, ctx) => {
    if (!(await ctx.guard.isSubscribed())) return;

    // Build context for Eve to reason over.
    const context = await getUserContext(ctx, user);

    // Start an Eve HITL session. Eve deliberates (possibly with human approval)
    // and POSTs its decision back to /v1/webhooks/eve when done.
    await startEveSession({
      agent: "retention-strategist",
      userId: user.id,
      callbackEvent: Events.AGENT_COMPLETED,
      input: {
        userId: user.id,
        email: user.email,
        recentEvents: context.events.slice(0, 20),
        emailEngagement: context.email,
      },
    });

    // Park until Eve signals completion (or 7 days pass).
    const result = await ctx.waitForEvent({
      event: Events.AGENT_COMPLETED,
      timeout: days(7),
      label: "eve-agent-result",
    });

    if (result.timedOut) return; // Eve didn't respond in time — leave them be.
    if (!(await ctx.guard.isSubscribed())) return; // unsubscribed during wait

    // Parse Eve's typed play from the event property.
    let play: { action: string; offerHeadline?: string; offerDetail?: string };
    try {
      play = SavePlay.parse(JSON.parse(String(result.properties?.play ?? "{}")));
    } catch {
      return; // malformed play — log + exit cleanly
    }

    if (play.action !== "send_offer") return;

    await sendEmail({
      to: user.email,
      userId: user.id,
      journeyStateId: user.stateId,
      template: Templates.CHURN_SAVE,
      subject: play.offerHeadline ?? "We'd love to keep you",
      props: {
        name: user.email.split("@")[0] ?? "there",
        offerHeadline: play.offerHeadline,
        offerDetail:   play.offerDetail,
      },
      journeyName: user.journeyName,
    });
  },
});
```

`examples/eve-agent/` in this repo contains a minimal standalone Eve service you can deploy separately (Node ≥ 24). It implements a `retention-strategist` agent that runs the HITL deliberation and POSTs its signed result back to `/v1/webhooks/eve`. It's documented as a separate deploy — not part of the turbo build.

## Why `waitForEvent` is the right primitive

`ctx.waitForEvent` is durable — the journey is checkpointed while it waits. The Hogsend worker can restart, redeploy, or crash, and the wait resumes cleanly from checkpoint. Eve's callback arrives whenever Eve finishes (seconds or days), and the journey wakes up and continues from exactly where it paused.

The 7-day timeout is the fail-safe: if Eve never responds (outage, misconfiguration), the journey exits cleanly with no send. The user is not permanently stuck.

<Callout type="info">
  - **`exitOn: [REACTIVATED]` fires immediately.** If the user reactivates (pays, returns, etc.) while the journey is parked on the wait, Hogsend cancels the Hatchet run — no post-wait branch executes, no save email fires.
  - **`AGENT_COMPLETED` must NOT be in `exitOn`.** The journey reads it via `waitForEvent`. If it were in `exitOn`, Hogsend would cancel the run when the event arrives — the post-wait branch would never fire.
  - **Event properties are scalars.** `eveSource.transform()` serialises the play as a JSON string in `eventProperties.play`, and the journey parses it back. This avoids the scalar-only constraint on event properties while keeping the data portable.
  - **Signature verification is fail-closed.** An unset `EVE_WEBHOOK_SECRET` means the `eve` webhook source is never mounted. A mismatched signature returns a `401`. There is no "enabled but unconfigured" state.
</Callout>
