Interactive local onboarding for a scaffolded Hogsend app. It mirrors the "next steps" `create-hogsend` prints after scaffolding, running them for you:

1. `docker compose up -d` — Postgres, Redis, and Hatchet-Lite
2. `cp .env.example .env` (if `.env` is missing)
3. Generate a `BETTER_AUTH_SECRET` (if it's still the placeholder)
4. `pnpm db:migrate` — the engine track, then your client track

```bash
hogsend setup [--cwd <dir>] [--yes] [--json]
```

## Options

| Option | Description |
|---|---|
| `--cwd <dir>` | Project root to run in (defaults to the current directory). |
| `--yes`, `-y` | Skip confirmation prompts (assume yes). Implied by `--json`. |
| `--json` | Run non-interactively and emit a single JSON result document. |
| `-h, --help` | Show this help. |

Run [`hogsend doctor`](/docs/cli/doctor) afterwards to verify the instance is healthy.
